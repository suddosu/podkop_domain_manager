# Podkop Domain Manager

Расширение для Chromium-браузеров для управления доменными адрес-листами
[podkop](https://github.com/itdoginfo/podkop) на роутере OpenWrt через **ubus**
(JSON-RPC поверх HTTP). Позволяет собирать домены прямо со страниц, держать их в
отдельных файловых списках, подключать списки к секциям podkop и проверять
пересечения между активными списками.

Без SSH в повседневной работе: разовая подготовка роутера по инструкции, дальше всё
делается из попапа расширения.

## Требования

- OpenWrt 24.10, podkop **≥ 0.7** (тестировалось на 0.7.19, sing-box 1.12.x).
- Пакеты на роутере: `uhttpd-mod-ubus`, `rpcd`, `rpcd-mod-file`.
- Любой браузер на Chromium (Chrome, Edge, Brave, Vivaldi …).

> Схема `/etc/config/podkop` менялась между версиями. Имена опций не захардкожены —
> мастер первого запуска читает живой `uci show podkop` и подставляет реальные.

## Что умеет

- **Секции** — показывает все routing-секции podkop, их community-, inline- и
  файловые списки. Сетевые интерфейсы, `connection_type` и прокси-настройки —
  только для чтения (жёсткий whitelist на запись).
- **Файловые списки** — отдельные plain-text файлы доменов в `/etc/podkop-lists/`
  (по домену в строке), подключаемые к секции через опцию `local_domain_lists`.
  podkop читает их с диска напрямую: без скачивания и без влияния
  `download_lists_via_proxy`. Создание, правка, удаление, привязка/отвязка.
- **Сбор доменов** — отдельная вкладка: «Перезагрузить и собрать» все домены,
  которые тянет активная страница (по `webRequest`); группировка по
  регистрируемому домену; выбор всех/части; добавление в существующий или новый
  список с проверкой покрытия активными списками перед добавлением.
- **Пересечения** — проход по всем секциям, поиск доменов, встречающихся более чем
  в одном активном списке. Содержимое community-списков резолвится из исходников
  репозитория `itdoginfo/allow-domains` (см. ниже).
- **Очередь изменений** — все правки копятся и применяются одним `reload`
  (чтобы не дёргать туннель на каждый домен). Перед применением — бэкап, есть «Откат».
- **Мастер первого запуска** — подключение, автоопределение схемы podkop, каталог
  списков, команда применения. Команды для роутера показываются прямо в мастере
  и на странице опций (с копированием).

## Установка

### 1. Роутер

Подробно — в [`/router-setup.md`](/router-setup.md). Кратко:

1. `opkg install uhttpd-mod-ubus rpcd rpcd-mod-file`
2. Завести отдельного пользователя `rpcd` с узким ACL.
   **Пароль только хэшем** (`uhttpd -m`), plaintext rpcd не принимает — это частая
   причина `PERMISSION_DENIED` при логине.
3. `mkdir -p /etc/podkop-lists`

Готовый скрипт со всеми командами можно скопировать прямо из расширения
(мастер первого запуска → «Команды для роутера», либо страница опций).

### 2. Расширение

1. `chrome://extensions` → включить «Режим разработчика».
2. «Загрузить распакованное расширение» → выбрать папку проекта.
3. В попапе пройти мастер: адрес роутера, пользователь, пароль (в открытом виде —
   rpcd сам хэширует и сравнивает).

## Команды применения

- **reload** — перечитать UCI и пересобрать конфиг sing-box без полного останова;
  подхватывает привязки и локальные файловые списки. По умолчанию.
- **restart** — полный stop+start: nftables/маршрутизация/sing-box с нуля. Надёжнее,
  но туннель на пару секунд полностью падает.
- **list_update** — только обновить содержимое удалённых/community-списков
  (скачивание). Правки локальных файлов и привязок этим НЕ применяются.

## Как резолвятся community-списки

podkop подключает community-теги как remote rule_set sing-box: бинарные `.srs` из
релизов `itdoginfo/allow-domains`, которые кэшируются самим sing-box (`cache.db`) —
на диске отдельных файлов нет, через `file.read` их не прочитать. Поэтому для
пересечений расширение тянет **plain-text исходники** тех же тегов из репозитория:

- `Categories/*.lst` — block, news, porn, anime, geoblock, hodca
- `Services/*.lst` — twitter, tiktok, discord, telegram, google_ai, google_play, hdrezka, youtube …
- `Russia/inside-raw.lst`, `Russia/outside-raw.lst` — russia_inside / russia_outside

Подсетевые теги (cloudflare, cloudfront, digitalocean, hetzner, ovh) — это IP-листы,
для доменных пересечений помечаются как `subnet`. Источник — ветка `main`, может
незначительно отличаться от скомпилированного релиза. Кэш 24 часа, принудительное
обновление — кнопкой «↻ community с GitHub».

Если `raw.githubusercontent.com` режется провайдером, фетч может не пройти —
соответствующий тег пометится как `net:…`.

## Безопасность

- Жёсткий whitelist UCI на запись: только опция файловых списков
  (`local_domain_lists`). Интерфейсы и прокси не правятся.
- Пароль rpcd хранится в `chrome.storage.local` без шифрования — используйте
  отдельного пользователя с узким ACL.
- Деструктивные действия (удаление списка, применение, откат) — с подтверждением.

## Структура

```
manifest.json          MV3
background.js          сборщик доменов (webRequest по активной вкладке)
lib/ubus.js            ubus JSON-RPC: login/uci/file/exec, chunked write, авто-релогин
lib/podkop.js          адаптер схемы podkop (v0.7+), файловые списки
lib/community.js       резолв community-тегов из allow-domains
lib/domains.js         нормализация, eTLD+1, покрытие, пересечения
lib/setup.js           генератор команд настройки + описания reload-команд
popup/                 UI: Секции / Списки / Сбор / Пересечения, мастер
options/               адрес роутера, креды, схема, команды настройки
docs/router-setup.md   подготовка роутера
```

## Ограничения

- Версии podkop < 0.7 не поддерживаются (другая схема UCI).
- Полнота сбора зависит от `webRequest`: домены, к которым страница не обращалась
  при загрузке, не попадут — используйте «Перезагрузить и собрать».
- eTLD+1 считается по компактному встроенному списку суффиксов (не полный PSL);
  при необходимости подставьте полный PSL в `lib/domains.js`.
