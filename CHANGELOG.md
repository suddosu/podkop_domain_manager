# Changelog

## 0.5.0
- Исправлена гонка двух перезапусков, ломавшая nftables. Причина: rpcd-метод
  `uci commit` сам шлёт `service event config.change`, а у podkop триггер
  `config.change → restart`; плагин дополнительно делал свой `restart` — два процесса
  параллельно, второй flush сносил nft-таблицу во время старта первого. Теперь после
  коммита плагин не делает отдельный restart: podkop перезапускается ровно один раз
  (как при Save&Apply в LuCI). Селектор команды применения убран как вводящий в заблуждение.
- Добавлена вкладка «Проверки»: диагностика podkop из плагина (get_status,
  get_sing_box_status, check_nft_rules, check_sing_box, check_dns_available, check_fakeip)
  через exec `/usr/bin/podkop` — ACL уже это разрешает.

## 0.4.0
- Команда применения по умолчанию изменена с `reload` на `restart`: podkop применяет
  правки своего UCI через restart (procd config.change → restart). `reload` в части
  сетапов оставлял лишние nft-правила маркировки — диагностика не проходила, требовался
  ручной перезапуск. Сохранённый `reload` автоматически мигрирует в `restart` один раз.
- Уточнены описания reload / restart / list_update.

## 0.3.0

- Подготовка репозитория: README, LICENSE (MIT), .gitignore, CHANGELOG.
